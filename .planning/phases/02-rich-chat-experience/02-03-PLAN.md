    ```typescript
    export function createChatHandler(

    ```

    If `settings` is undefined, use defaults: `{ toolVisibility: 'verbose', interruptionBehavior: 'abort' }`.

    **Handler function body** (replacing the current implementation):

    1. **New session detection** (unchanged from Phase 1 — D-05):
       `if (context.history.length === 0)` -> `await processManager.restart().catch(() => {});`

    2. **Lazy start** (unchanged from Phase 1 — Gap 1 closure, D-05):
       `getState()` -> check sessionId -> show `'Starting Pi...'` progress if null -> `await processManager.start();`

    3. **Working progress**: `stream.progress('Pi is working...');`

    4. **Subscribe to events BEFORE prompt and create completion promise** (Pitfall 1 — CRITICAL):
       The `onEvent` listener processes each event through `mapAgentEventToAction` / `applyStreamAction`, AND detects agent completion by checking `event.type === 'agent_end'`. This replaces `waitForIdle()` which is not exposed on the `PiProcessManager` interface (the underlying `RpcClient` has it but the wrapper does not).
       - Declare: `let resolveCompletion: (() => void) | null = null;`
       - Create: `const onComplete = new Promise<void>((resolve) => { resolveCompletion = resolve; });`
       - Subscribe: `const unsubscribe = processManager.onEvent((event) => { mapAgentEventToAction(...); applyStreamAction(...); if (event.type === 'agent_end') resolveCompletion?.(); });`

    5. **Send prompt verbatim** (D-07 passthrough):
       `await processManager.prompt(request.prompt);`
       `request.prompt` already excludes the `@pi` participant prefix. Sent verbatim to Pi — no parsing, no inspection.

    6. **Race completion against cancellation** (Pitfall 4 prevention, D-05):
       Create `AbortController`. Wire `token.onCancellationRequested(() => abortController.abort())`.
       `await Promise.race([onComplete, rejectOnAbort])` where `rejectOnAbort` is a Promise that rejects with `CancellationError` when `abortController.signal` fires.

    7. **Catch cancellation and handle interruption**:
       If `err instanceof CancellationError`:
       - `abort` behavior: `await processManager.abort();`
       - `followUp` behavior: do nothing (next prompt delivered via new handler invocation)
       - Return `{}`
       If NOT CancellationError: throw (re-throw for outer catch).

    8. **Finally block**: `unsubscribe()` to clean up onEvent listener.

    9. **Outer catch** (process crash — unchanged from Phase 1):
       `stream.markdown('**Pi process exited unexpectedly.**\n\n```\n' + error + '\n```\n\nSend another message to restart.')` -> return `{}`

    **Nesting structure** (two try/catch levels):
    ```
    try {                    // OUTER: catches process crash
      // history check + restart
      // lazy start + progress
      try {                  // INNER: catches cancellation
        // onEvent subscription + onComplete promise
        // prompt()
        // race: onComplete vs CancellationError
      } catch (err) {
        if CancellationError => abort/followUp; else throw
      } finally {
        unsubscribe()
      }
      return {};
    } catch (err) {
      stream.markdown(error message);
      return {};
    }
    ```

    **Remove** the old `streamEvents` import — replace with `mapAgentEventToAction` and `applyStreamAction`.
    Indentation: 2 spaces (vscode-ext convention).
