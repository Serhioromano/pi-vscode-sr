    **Step 2: Write vscode-ext/src/pi-process-manager.ts (GREEN)**
    2-space indent, single quotes. Follow the factory closure pattern exactly:

    ```typescript
    import { RpcClient } from '@earendil-works/pi-coding-agent/dist/modes/rpc/rpc-client.js';
    import type { AgentEvent } from '@earendil-works/pi-agent-core';

    export interface PiProcessManagerState {
      client: RpcClient | null;
      cwd: string;
      model?: string;
      provider?: string;
      sessionId: string | null;
    }

    export interface PiProcessManager {
      start(): Promise<void>;
      stop(): Promise<void>;
      restart(): Promise<void>;
      prompt(message: string): Promise<void>;
      promptAndWait(message: string, timeout?: number): Promise<AgentEvent[]>;
      abort(): Promise<void>;
      onEvent(listener: (event: AgentEvent) => void): () => void;
      getState(): Promise<{ sessionId: string | null }>;
    }

    export function createPiProcessManager(opts: {
      cwd: string;
      model?: string;
      provider?: string;
    }): PiProcessManager {
      const state: PiProcessManagerState = {
        client: null,
        cwd: opts.cwd,
        model: opts.model,
        provider: opts.provider,
        sessionId: null,
      };
      const listeners = new Set<(event: AgentEvent) => void>();

      return {
        async start() {
          if (state.client) return; // Already started
          state.client = new RpcClient({
            cwd: state.cwd,
            provider: opts.provider,
            model: opts.model,
          });
          await state.client.start();
          // Per Open Question 1: call start() then optionally newSession()
          // For now: just start, then read state
          const sessionState = await state.client.getState();
          state.sessionId = sessionState.sessionId;
          // Forward events from RpcClient to our listeners
          state.client.onEvent((event: AgentEvent) => {
            for (const listener of listeners) {
              listener(event);
            }
          });
        },
        // ... other methods
      };
    }
    ```

    Key design decisions per D-03, D-05, D-06:
    - **Lazy allocation**: RpcClient is created inside `start()`, not in the factory constructor. This enables lazy start (D-05).
    - **Crash handling**: If the Pi process exits, RpcClient emits 'exit' events or rejects promises. The `prompt()`/`promptAndWait()` methods should catch and propagate errors. The caller (chat-handler.ts) handles crash visibility per D-06.
    - **Workspace isolation**: `getState()` returns session info. `switchSession()` will be called by extension.ts on workspace switch (D-08) — the method is available on the underlying RpcClient, exposed via the manager.
    - **No error in factory**: The factory constructor never throws. Errors only come from async methods (start, prompt, etc.).

    Method implementations:
    - `stop()`: If client exists, call `state.client.stop()`, set client to null
    - `restart()`: Call stop() then start()
    - `prompt(message)`: If client exists, call `state.client.prompt(message)`
    - `promptAndWait(message, timeout?)`: If client exists, call `state.client.promptAndWait(message, timeout)`
    - `abort()`: If client exists, call `state.client.abort()`
    - `onEvent(listener)`: Add to listeners Set, return unsubscribe function `() => listeners.delete(listener)`
    - `getState()`: If client exists, return client state; otherwise `{ sessionId: null }`
