## 6. Runtime Directories

### 6.1 `.pi/` — Inter-Process Communication

The `.pi/` directory is the filesystem-based IPC channel between the Pi agent and VS Code extension:

```
.pi/
├── review-requests/       # Agent writes review request JSON files here
├── review-results/        # VS Code extension writes review results here
├── npm/                   # Pi agent runtime dependencies (has own lockfile)
│   └── package-lock.json  # Runtime dependency lockfile
└── .vscode-ready           # Heartbeat timestamp file
```

- **Not committed to git** (should be in .gitignore)
- Created at runtime by both processes
- Communication protocol: agent writes request → extension picks up → extension writes result → agent reads result

### 6.2 `.vscode/` — VS Code Workspace

Standard VS Code workspace configuration directory — settings, launch configs, extension recommendations.
